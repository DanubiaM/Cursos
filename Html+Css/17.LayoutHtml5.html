Layout em HTML 5

Antigos Layouts

Até essa sessão do livro, foi visto o uso de div para realizar a divisão das partes de um site, como por exemplo uma div com id identificando a pagina, header, nav, content, article, footer... Entre outras diversas partes de um site. Com o HTML5 surgiu novas tags no HTML.

Novo Layout

O HTML5 nos trouxe novas tags para facilitar a leitura de uma pagina. Mecanismos de pesquisas por exemplo tem a facilidade de procurar conteudo em uma pagina devido o uso dessas novas tags. Enfim, qual é a mudança? Anteriormente vimos o uso da div com seu id para dividir a pagina, agora não é necessário uma div para realizar essa separação. Por exemplo, uma página sempre contem uma header dessa maneira teremos então uma tag com o nome header e frequentemente encontramos uma barra de navegação dentro da header, logo temos também uma nova tag, a nav. Além dessas duas novas tags, temos a tag article, aside e footer.

Headers e Footers

As tags header e footer podem ser usadas em um contexto geral da pagina, ou seja, para criar o cabeçalho e rodapé da pagina, ou então dentro de um artigo ou sessão (tags article e section). Dentro de uma header frequentemente encontramos o logo e a barra de navegação, e para artigos pode ser usado para conter o titulo e a data. O footer encontramos detro dele termos de uso, copyright, polita de privacidade ou links rapidos, já para artigos podemos ter também referenciar bibliograficas dentro da tag footer.

Navegação

A tag nav contem o principal bloco de navegação da pagina. Para a criação da barra de navegação, o funcionamento é o mesmo que o uso da div com id nav e com lista de itens dentro, so muda que não utiliza mais div e sim a tag nav.

Artigos

A tag article funciona como um container para qualquer sessão de uma pagina, podendo estar sozinha um junta. Podendo ser um artigo individual ou blog, um comentario ou post de um forum, ou qualquer outro elemento independente. Debtri de um article podemos ter diversos itens, figuras, titulos e subtitulos, paragrafos.

Aside

A tag aside é usada com dos propósitos, quando esta dentro da teg article ou não. Quando a mesma se encontra dentro da tag article, ele pode conter informações que é relatado no article, mas não essencial, como por exemplo glosário e citação.
Quando esta fora da tag article, ele funciona como um container para conteudo relacionado a pagina. Como por exemplo, links de outras sessões, lista de poster recentes, box de procura, tweets recenter da pagina...

Sessões

A tag section é usada para manter junto um conteudo, cada sessão pode haver mais de uma heading. Por exemplo, em uma pagina, podemos ter varias section, cada uma contendo um conteudo diferente, tal como top produtos, mais vendidos, etc.
Não deve ser usado para substituir a div e nem estruturar uma página.

Grupo de títulos

A tag hgroup é usada para agrupar um conjunto de titulos, desde o h1 à h6. Podendo ser usada então como um conjunto de titulo de subtitulo. Alguns desenvolvedores não gostam de utilizar pois fazem o subtitulo usando a tag p.

Figuras

A tag figure pode ser usada para conter qualquer conteudo que referencia o fluxo principal de um artigo, não apenas uma imagem. Logo podemos usar a tag quando temos: imagens, videos, graficos, diagramas, code.

Linkando um bloco

Vimos até então o uso da tag a para criar links, sabemos até então que é uma tag sem fechamento. Porem apartir de agora podemos ter um fechamento para a tag, e dessa maneira linkar todo um bloco de elemento.
