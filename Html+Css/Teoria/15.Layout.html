Construindo Blocos

O CSS trata os elemetos em HTMl como se fosse um unico box. Esses boxs podem ser block-level ou um inline box. Um block-level funciona como um construtor de blocos, como por exemplo h1, p, ul, li, div. Já o inline funciona ao lado dos textos do bloco, como por exemplo img, b, i, strong, em.
Se um block-level contem dentro dele um outro block-level, então eles serão parentes, isso acontece comumente quando utilizamos div.

Controlando a posição dos elementos

No CSS utiliza-se o controle de posição para controlar o layout de uma pagina. Ela pode conter os seguintes valores: normal flow, relative posição e absolute posição. Para isto, basta que se especifique o valor da propriedade position

Normal Flow
O padrão normal de paragrafo
Relative positioning
Move o elemento para uma posição especifica, como top, right, bottom e left. Ele não afeta os demais elementos.
Absolute positioning
Posiciona o elemento em relaçao ao conteudo do elemento. Não afeta a posição de qualquer elemento envolta dele. Posiciona o elemento para se mover junto com o scroll do mouse.
Fixed positioning
É uma forma de posicionamento absoluta sobre a janela do navegador. Não se move quando o usuario meche o scroll do mouse e nem afeta outros elementos.
Floating elements
Aceita colocar um elemento do normal flow e posicionar à direita ou esquerda do box.

Normal Flow

No normal flow,cada elemento do block-level fica encima do proximo. Ele é o caminho padrão que o navegador trata os elementos Mas ainda sim podermos definir sua sintaxe como position:static

Relative positioning

A posição relativa move os elementos em relação aonde estaria o normal flow. A partir do momento que utilizamos essa propriedade podemos mover o objeto para varias direções, usando as propriedades left, top, bottom, right e especificando o valor em px.

Indicamos seu uso utilizando o position:relative(aqui estou utilizando além da relative positioning o top e left.)

Absolute positioning

A utilizarmos a posição absoluta o texto sera retirado do normal flow(padrão) e não afetara a posição dos demais elementos. Apartir de então podemos move-lo utilizando as propriedades left, top, bottom, right e especificando o valor em px.

Fixed positioning

Utilizando a propriedade position:fixed temos então um elemento fixado na janala do navegador. Isso quer dizer que, mesmo o usuario movento a tela. Observe o gato

Sobreposição de elementos

z-index

O z-index funciona como um empilhamento, como por exemplo na area de design, quando vamos utilizar camadas podemos trazer para frente ou trazer para tras. O seu valor pode ser numeros, sendo positivo para frente do elemento e negativo para baixo do elemento. O valor deste numero depende de outros valores que tenha utilizando o z-index, se possuir um elemento utilizando z-index:1 e querer que tenha outro elemento sobreposto a ele basta que esse novo elemento possua z-index:2.

Flutuando elementos

float

A propriedade float realiza uma flutuação do elemento para a direção especificada, que corresponde ao seu valor, left, top, bottom, right. Todos os elementos que estão contidos dentro do box também são deslocados juntos. Podemos utilizar essa propriedade, por exemplo em um blockquote, deslocando-o para a direita.

Por causa da variação do tamanhos das telas, frequentemente designers utilizam o tamanho de 960-1000px. Eiste dois tipos de largura de layout: fixed e liquid.

Layoutes de largura fixada

Este tipo de layout não muda de tamanho se o usuario aumenta ou diminiu a tela. As medidas tendem a ser dadas em pixels. Sendo assim, para criarmos esse layout a largura dos boxes devem ser especificados em pixels.

Layout em Grids

Esse tipo de layout é frequentemente utilizado por designers. A cada possui 960 pixels. Podemos ver sua aplicação em site de noticias, ou algum q contenha bastante informação.
Existe varios tipo de grids as com 960 pixels de largura e com 12 colunas, 16 colunas, ou menos como 8 ou 4, dependendo do tipo de dispositivo de ira ser utilizado. No tipo de grid com 12 colunas nos temos que, cada coluna possui uma margem de 10 pixels, o vão entre elas é de 20 pixels e 10 pixels da esquerda e titulo da página.

Framework: 960.gs Grid

É um dos frameworks mais utilizados para criação de layouts de paginas. É utilizado uma nomeoclatura de classes para cada coluna. Utilizando um layout de 960pixels, temos 12 colunas, logo cada uma das colunas deve ter seu nome em uma classe, por exemplo container_1. E cada bloco contido na coluna deve ter o nome de grid_3 por exemplo.

Multiplos stylesheet

Utilizamos essa propriedade para podermos importar paginas diferentes do CSS, por exemplo podemos ter um CSS para o layout em uma pagina e o CSS para a barra de navegação em outra. Existe dois caminhos para isto, utilizando o @import no CSS ou link no HTML

@import

Para utilizarmos essa propriedade basta que no inicio do documento CSS, utilizemos a seguinte propriedade @import url("endereço.css"). Dessa maneira podemos importar varios documentos CSS.

link

Para criarmos vairos stylesheet utilizando o link basta que adicionamos varios links do documento css, da mesma maneira que é feito com um unico documento CSS. Exemplo: link rel="stylesheet" type="text/css" href="css/cite.css e link rel="stylesheet" type="text/css" href="css/layout.css É importante notar que a primeira que a regra do primeiro documento CSS vai valer primeiro, pois vem antes do proximo documento.
